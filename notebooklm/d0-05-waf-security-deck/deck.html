Five security design principles

Plan · Confidentiality · Integrity · Availability · Sustain

Grounded · Defender for Cloud

Where the principles surface in practice

Plan readiness · protect confidentiality (least privilege, classify, encrypt) · protect integrity (supply chain, immutable backups) · protect availability (resource exhaustion, recovery rigor) · sustain the posture (threat modeling, testing).

Each principle is driven by Zero Trust — verify explicitly, least privilege, assume breach.

A lifecycle path, not a scoreNETWORK INTELLIGENCE · FOUNDATIONS
The five-level security maturity model

Sequence the work — move up one level at a time

1

Core security

Minimum viable posture — the baseline. Externalize identity, encrypt data, protect secrets.

2

Threat prevention

Secure deployment, classify data, apply authN/authZ, harden the attack surface.

3

Risk assessment

Threat modeling, classify flows, advanced encryption, an incident response plan.

4

System hardening

Refine with production insights — microsegmentation, JIT, tighter IAM.

5

Advanced defense

Enterprise-grade — DDoS, DLP, confidential computing, SIEM/SOAR.

Not a compliance score.  Place the workload honestly, then advance one level — each level carries explicit tradeoffs.

Every control carries a tradeoff

MCSB tells you the control; the Well-Architected Framework tells you what designing it in costs the rest of the workload. Microsegmentation adds cost and complexity; bring-your-own-key adds key-management burden.
