# AZ-500 Azure Security Course Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce ~21 narrated Azure-portal training videos covering AZ-500, in `out/az500-final/`, using the humboldt pipeline — provisioning a per-domain lab, recording the real portal, branding, with exam framing + hands-on labs.

**Architecture:** Reuse the existing author-JSON → `build.mjs` (TTS+record) → redact → `brand.mjs` → place pipeline (proven on SCS-C02, cloud-agnostic). Add Azure portal browser auth, per-domain Bicep/`az` labs (build→record→teardown), corporate read-only Entra recording with column redaction, and a rendered-diagram module type for the D0 frameworks. All authoring grounded in the Microsoft Learn MCP.

**Tech Stack:** Node 20 + Playwright (recorder), ffmpeg, OpenAI TTS (`gpt-4o-mini-tts`, voice **`sage`** for this series — the AWS/SCS-C02 course stays `alloy` so the two series are audibly distinct), Azure CLI + Bicep, Microsoft Learn MCP (`plugin:microsoft-docs:microsoft-learn`).

**Spec:** `docs/superpowers/specs/2026-06-13-az500-sc100-course-design.md`

## Per-module production recipe (apply to every portal module)

1. **Ground:** `microsoft_docs_search` (+ `microsoft_docs_fetch` on the top hit) for the service's security guidance and exact portal blades. Never invent a blade route.
2. **Author** `modules/az-<id>.json` from the `aws-iam.json` template: `cloud:"azure"`, **`voice:"sage"`**, the domain's `examDomain`, 5 sections (each a verified portal `url` + kicker + cardTitle + 3 bullets + natural-language narration), one `drill`, recap (5 rules + examTip), action (2 labs). Omit `redactions` initially.
3. **Validate:** `node -e "const j=require('./modules/az-<id>.json');import('./lib/schema.mjs').then(m=>{m.validateModule(j);console.log('OK')})"` → `OK`.
4. **Build:** `caffeinate -dimsu node bin/build.mjs az-<id>` (window frontmost).
5. **Verify:** streams present (`ffprobe`), real speech (`volumedetect`), 3–4 frames inspected → real blades + overlays, ≤8 min.
6. **Redact:** internal-light for personal-lab modules (chrome + any subscription/resource IDs); **full column/field redaction for corporate Entra** with KK approval.
7. **Brand + place:** `node bin/brand.mjs az-<id>` → copy `*-final.mp4` to `out/az500-final/NN-dX-<id>.mp4`.
8. **Commit** the module JSON; tick it in `plans/az500-build.md`.

## Verification gates (the "tests" for this pipeline)

- Schema: `validateModule` returns OK (and `node --test` stays green for any toolkit code change).
- Recording: `ffprobe` shows video+audio; `volumedetect` shows speech; extracted frames show the intended blade + overlay (not sign-in/splash).
- Length: every final ≤ 480 s.
- Privacy: corporate-Entra frames show no legible PII after redaction (multi-timestamp check + KK approval).
- Cost: after each domain, `az group show`/`az security pricing list` confirm teardown — no lingering paid meters.
